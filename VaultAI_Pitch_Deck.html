The Problem

🔒 The Trust Gap
People want AI to manage their money — but fear uploading raw bank statements to the cloud. One breach and your entire financial life is exposed. No one should have to choose between convenience and privacy.
🐟 Goldfish Memory
AI financial bots forget you the moment you close the tab. Real planning needs 365 days of context — remembering you had a baby in February to surface a $2,000 tax credit in April.
📊 Fragmented Tools
Tax software, budgeting apps, and investment platforms don't talk to each other. You become the integration layer — copying numbers between tabs while your money sits idle.

The Solution

A Family Office in your pocket — running entirely on your local machine.
🛡️ Sanitizes Data
Strips PII before the AI sees anything. Account numbers, emails & phone numbers become ACCT_001, EMAIL_001 — locally, instantly, before any network call.
🧠 Builds Memory
Extracts life events from casual chat and persists them to a local vault. SQLite for exact queries. ChromaDB for semantic recall. 365 days of context — never touching the cloud.
⚡ Executes Strategy
Runs deterministic cash-flow forecasting, searches live tax codes via Tavily, and recommends exact financial actions with sources — like a CFO on demand.

System Architecture

Layer 1: PII Masker
Regex strips account numbers, emails, SSNs, phone numbers → stable UIDs. No network access.
LOCAL
↓ masked data only
Layer 2: Memory Retrieval
SQLite pulls life events & conversation history. ChromaDB semantic recall finds relevant past context.
LOCAL
↓ context package (masked)
Layer 3: LLM Agent
Claude Haiku sees only masked data + context. Decides which tool to call. Raw data never reaches here.
CLOUD ☁️
↓ tool result
Layer 4: Tool Execution
Forecaster, memory writer run locally. Tavily is the only outbound exception — for live tax & rate data.
LOCAL*
↓ de-mask → user
* Tavily search is the only outbound call besides the LLM itself.

Privacy — Not a Policy, a Guarantee

🔒 Raw data never leaves your machine
The PII masker has no network access. Docker enforces isolation at the network layer — it is physically impossible for raw data to reach the cloud. This is architecture, not a promise.
🔑 Stable UID mapping within a session
"john.doe@gmail.com" always becomes EMAIL_001 for the duration of a session. The lookup table lives only in RAM — cleared when the server restarts. No persistence of PII anywhere.
👁️ What Claude actually sees
Only masked CSV text + your typed messages. No names, account numbers, phone numbers, SSNs, or email addresses ever reach the Anthropic API. UIDs are swapped back after Claude responds.
"The local-first architecture is our regulatory moat — we cannot comply with a data request for data we do not hold."

Roadmap to Scale

Multi-device sync
Optional encrypted cloud backup — server holds blobs it cannot read (client-side encryption). Privacy preserved at scale.
Multi-user
Swap SQLite → PostgreSQL (one-line SQLAlchemy config change). Add auth layer. Architecture already supports it.
Better PII masking
Replace regex with spaCy NLP model for name & entity detection. Catch PII that patterns miss.
Zero-knowledge cloud
PII masker runs in a sandboxed container with no network access — only masked context ever reaches any server.
Task execution
Agent moves from giving advice → executing tasks: tax filing, investment allocation. Requires regulatory review.
