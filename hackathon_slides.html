The Problem

🔓

The Trust Gap

People want AI to help with their finances — but uploading raw bank statements to the cloud means handing your most sensitive data to a third party.

🐟

Goldfish Memory

Current AI assistants forget everything when you close the tab. Real financial planning requires months of context — your life events, income history, and goals.

The Solution

🔒

Private by architecture

PII is masked locally before any data reaches the cloud. The LLM only reasons on sanitized data — never raw account numbers or personal info.

🧠

Persistent memory

SQLite stores structured history. ChromaDB enables semantic recall. Ask about a tax credit in April — it surfaces the relevant event from months ago.

🔍

Live context

Tavily search brings in real-time market data, tax codes, and interest rates — grounding the AI in current reality, not training data.

How It Works

1

Claude Haiku reasons on sanitized context. Decides which tool to call.

forecast_cashflow (local) · search_web (Tavily) · save_life_event (local)
↓
5
De-masking — local
UIDs swapped back to original values before the user reads the response.

Problems We Solved

Problem
Docker build took 30+ minutes. sentence-transformers silently pulled 2GB CUDA PyTorch.
Solution
Pre-install CPU-only torch before requirements.txt. Build time: 30 min → 5 min.
Problem
Build context was 670MB. Docker was copying the entire venv/ folder on every build.
Solution
Added .dockerignore to exclude venv/, chromadb/, node_modules/. Context: 670MB → 78KB.
Problem
Date formats were inconsistent in SQLite — "April 2026", "04/24/2026", "2026-04-27" mixed together. Sort order was wrong.
Solution
normalizeDate() with dateutil.parser standardizes all dates to YYYY-MM-DD before saving.

Tradeoffs

SQLite over PostgreSQL
Zero setup, embedded, no extra container. Tradeoff: no concurrent writes. Acceptable for a single-user local app.
In-memory CSV session
Fast and simple — CSV lives in a Python dict. Tradeoff: lost on server restart. Fine for a demo.
CPU-only PyTorch
Docker image is ~8GB lighter. Builds 6x faster. Tradeoff: slower embeddings. Adequate for short text on a local machine.
Naive cash flow forecast
Next month = this month's net. Fully deterministic — no LLM in the math. Tradeoff: not sophisticated. Benefit: zero hallucination risk on financial numbers.
Local embeddings
all-MiniLM-L6-v2 runs on device. No embedding API cost, no data sent out. Tradeoff: slightly lower quality than hosted models.
